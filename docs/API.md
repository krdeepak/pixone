# API Reference

Base URL: `http://localhost:8000/api`

Interactive docs (auto-generated by Django Ninja): `http://localhost:8000/api/docs`

All endpoints accept `multipart/form-data` (image upload). All responses are JSON.

---

## Reframe

### POST /reframe/

Manual reframe — crop an image to the specified rectangle.

**Request** — `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `image` | file | yes | Image file to crop |
| `x` | int | yes | Left edge of crop box (pixels) |
| `y` | int | yes | Top edge of crop box (pixels) |
| `width` | int | yes | Width of crop box (pixels) |
| `height` | int | yes | Height of crop box (pixels) |
| `output_width` | int | no | Resize output to this width |
| `output_height` | int | no | Resize output to this height |

**Response 200**

```json
{
  "request_id": 42,
  "output_url": "/media/results/reframe/42_photo.jpg",
  "metadata": {
    "original_size": [1920, 1080],
    "crop_box": [100, 50, 900, 650],
    "output_size": [800, 600]
  }
}
```

**Errors**

| Status | Meaning |
|--------|---------|
| 422 | Validation error (missing or invalid fields) |
| 500 | Processing failed |

### POST /reframe/smart/

Smart reframe — automatically crop around the most prominent face.

**Request** — `multipart/form-data`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `image` | file | yes | — | Image file to reframe |
| `mode` | string | no | `standard` | Crop tightness: `zoomed`, `standard`, or `full` |
| `aspect_ratio` | string | no | `1:1` | Target ratio: `1:1`, `4:3`, `16:9`, `9:16`, `3:4` |

**Modes**

| Mode | Padding | Description |
|------|---------|-------------|
| `zoomed` | 0.3× face size | Tight — face fills the frame |
| `standard` | 0.5× face size | Balanced — face + shoulders |
| `full` | 1.8× face size | Wide — shows scene context |

**Response 200**

```json
{
  "request_id": 43,
  "output_url": "/media/results/reframe/43_smart_photo.jpg",
  "face_count": 2,
  "mode": "standard",
  "metadata": {
    "original_size": [1920, 1080],
    "crop_box": [400, 100, 1200, 900],
    "output_size": [800, 800],
    "face_count": 2,
    "primary_face": {"x": 500, "y": 200, "w": 200, "h": 250},
    "mode": "standard",
    "aspect_ratio": "1:1"
  }
}
```

**Errors**

| Status | Meaning |
|--------|---------|
| 422 | No faces detected in the image |
| 422 | Validation error (invalid mode or aspect ratio) |
| 500 | Processing failed |

## Face Detection

### POST /face-detection/

Detect faces in an image and return bounding boxes with confidence scores.

**Request** — `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `image` | file | yes | Image file to analyse |

**Response 200**

```json
{
  "request_id": 44,
  "faces": [
    {
      "x": 500,
      "y": 200,
      "width": 200,
      "height": 250,
      "confidence": 99.7,
      "landmarks": null
    }
  ],
  "face_count": 1
}
```

`landmarks` is `null` unless the detector returns landmark data. When present, it is a dict of landmark name → `{x, y}` coordinates (normalised 0–1, relative to image size, as returned by Rekognition).

**Errors**

| Status | Meaning |
|--------|---------|
| 500 | Detection failed (AWS error, etc.) |

## Common Notes

- All coordinates (`x`, `y`, `width`, `height`) in responses are in **pixels**, absolute to the original image.
- `output_url` is a relative URL when using local storage. Prepend the backend base URL to construct the full URL.
- `request_id` maps to a `ProcessingRequest` row in the database. Viewable in Django admin at `/admin/`.
- Jobs execute synchronously — the HTTP response is returned after processing is complete.
